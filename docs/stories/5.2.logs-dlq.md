## Story

**As a** Lucas (operador),
**I want** uma página `/logs` com 2 abas — "Dead Letter Queue" e "Eventos não-mapeados" — onde reprocesso jobs em lote, expando histórico de retries e crio campanha a partir de tokens recebidos errados,
**so that** eu resolva incidentes em segundos sem mexer em código ou Redis.

## Acceptance Criteria

1. Rota `/logs` no painel com 2 tabs.
2. **Tab DLQ:**
   - Lista cards com worker chip + evento + token + erro + tempo
   - Botão "Detalhes" expande timeline dos 4 retries com timestamps e mensagens de erro
   - Botão "Reprocessar" individual reenfileira na fila origem com prioridade alta — toast de confirmação
   - Botão "Reprocessar todos (N)" no header — spread temporal (1 job por 200ms) pra evitar rate limit
3. **Tab Eventos não-mapeados:**
   - Lista cards com token recebido + payload JSON pretty-printed (syntax highlight)
   - Botão "Criar campanha →" abre modal pré-preenchido com o token
   - Botão "Descartar" remove do log
4. Busca no header da página filtra por token / evento / email / código de erro.
5. Endpoints backend:
   - `GET /api/dlq?worker=&q=` — lista DLQ agregada
   - `POST /api/dlq/:id/retry` — move job pra fila ativa
   - `POST /api/dlq/retry-all` — bulk retry com spread
   - `GET /api/unmatched-events?q=` — lista
   - `DELETE /api/unmatched-events/:id` — descarta
6. Fidelidade visual ao prototype `src/ui/logs.jsx`.
7. Empty state quando não há nada: ícone verde "✓" e mensagem.

## Dev Notes

**Referência visual:** `src/ui/logs.jsx` — replica fidedignamente, especialmente:
- Timeline de retries com bullet colorido (vermelho pra erros)
- Card de unmatched com syntax-highlighted JSON inline
- Botões com cor semântica certa (danger para descartar, primary para reprocessar)

**Bulk retry — anti-pattern:** NÃO usar `Promise.all` para retry de centenas de jobs simultaneamente — vira tempestade contra APIs externas. Spread temporal obrigatório:
```ts
for (const job of failed) {
  await job.retry();
  await sleep(200);
}
```

**DLQ no BullMQ:** Falhas terminais ficam no estado `failed`. `Queue.getFailed(start, end)` retorna eles. Reprocessar = `job.retry()`.
