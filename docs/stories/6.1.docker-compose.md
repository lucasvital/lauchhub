## Story

**As a** @devops responsável pela infra Loyola,
**I want** Dockerfile multi-stage + docker-compose.yml prontos pra subir no Coolify do VPS,
**so that** o backend LaunchHub (sem o frontend, que vai pro Vercel) rode em produção via UI do Coolify, com SSL e proxy gerenciados automaticamente.

## Acceptance Criteria

1. `Dockerfile` multi-stage:
   - Stage 1 (builder): `node:20-alpine` — `npm ci`, `npm run build`
   - Stage 2 (runtime): `node:20-alpine` — apenas `dist/`, `node_modules` produção, `package.json`
   - User não-root no runtime
   - `HEALTHCHECK` apontando pra `GET /health`
   - Imagem final < 250MB
2. `docker-compose.yml` na raiz com 3 serviços:
   - `launchhub` — backend Fastify (build via Dockerfile local)
   - `postgres` — `postgres:16-alpine`
   - `redis` — `redis:7-alpine`
3. **SEM labels Traefik no compose** — Coolify gerencia proxy via UI. Compose expõe a porta do app internamente; Coolify roteia.
4. Variáveis env injetadas via Coolify UI (todas as do `.env.example` da Story 1.1) — compose lê via `${VAR}` ou `env_file`.
5. Volumes nomeados (`postgres_data`, `redis_data`) com persistência (Coolify gerencia volumes).
6. Resource limits opcionais (`deploy.resources.limits` é ignorado fora de Swarm; documentar em comentário).
7. Health checks em todos os 3 serviços:
   - `launchhub`: HTTP `GET /health`
   - `postgres`: `pg_isready -U $POSTGRES_USER`
   - `redis`: `redis-cli ping`
8. `docker-compose.dev.yml` separado (criado na Story 1.1) permanece — sobe só Postgres + Redis localmente sem app.
9. `.dockerignore` adequado (exclui `node_modules`, `.env`, `.git`, `tests/`, `src/ui/**`, `docs/`, `.aiox-core/`, `.claude/`).
10. Build local funciona: `docker build -t launchhub:test .` produz imagem rodável.
11. Documentação em README: seção "Deploy via Coolify" com screenshots/passos.

## Dev Notes

**Mudança vs CLAUDE.md original:** O CLAUDE.md prevê Docker Swarm + Traefik labels manuais. Decisão arquitetural 2026-05-14: **trocar Swarm por Coolify** no VPS, que já está rodando e gerencia Traefik internamente. Resultado: compose mais simples, sem labels Traefik no arquivo, sem network externa, sem `deploy.resources` (ignorado).

**Frontend NÃO entra neste compose** — vai pro Vercel via Story 4.1 (Task 9). Este compose tem só backend + DB + Redis.

**Coolify Build Pack:** "Docker Compose" — Coolify detecta `docker-compose.yml` e builda automático. Alternativa "Dockerfile" funciona, mas Compose dá controle melhor sobre Postgres/Redis adjacentes.

**Persistência:** Coolify mapeia volumes nomeados pra paths gerenciados (geralmente `/var/lib/docker/volumes/`). Backup é responsabilidade da Story 6.2.

**SSL:** Coolify provisiona Let's Encrypt automático ao plugar o domínio na UI. Zero config no compose.

**Secrets:** Coolify UI tem campo "Environment Variables" — marca como secret (não logado nos builds). Não usar `docker secret` (esse é Swarm-specific).

### Testing

- Build local: `docker build -t launchhub:test .` deve completar < 2min
- Run local com env: `docker run --env-file .env -p 3000:3000 launchhub:test` → `/health` retorna 200
- Validação real é em produção (Story 6.2)
