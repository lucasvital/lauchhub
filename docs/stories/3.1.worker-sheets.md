## Story

**As a** gestor de marketing,
**I want** cada evento de campanha logado automaticamente numa Google Sheets configurada,
**so that** eu tenha um histórico legível sem precisar logar no painel.

## Acceptance Criteria

1. `src/workers/sheets.worker.ts` cria BullMQ Worker consumindo `queue:sheets`.
2. Cada job recebido apenda 1 linha na planilha cuja `sheets_id` está no payload.
3. Colunas na ordem **exata** definida em CLAUDE.md: `timestamp`, `event`, `name`, `email`, `phone`, `order_id`, `payment_method`, `value`.
4. Autenticação via service account JSON (`GOOGLE_SERVICE_ACCOUNT_JSON` do env, parseado).
5. Range usado: `Sheet1!A:H` (configurável por campanha futuramente, mas default Sheet1).
6. Se a planilha não tiver header, a primeira execução cria header. Subsequentes só apendam.
7. Retry policy do BullMQ: 4 tentativas com backoff `[0s, 30s, 5min, 30min]`. Após 4 → DLQ (`sheets:failed`).
8. Erros classificados:
   - 401/403 → falha terminal imediata (DLQ direto, não retry — credencial errada não resolve com retry).
   - 429 (quota) → retry com backoff exponencial.
   - 5xx → retry padrão.
9. Logs estruturados com `correlation_id` em cada etapa.
10. Teste de integração mockando a API do Google Sheets (`nock` ou similar) cobre: append OK, 429 retry, 401 fail-fast, append idempotente (mesmo job 2x não duplica → discutir estratégia: column G `order_id` como chave única? Decidir com @architect).

## Dev Notes

**Referência:** CLAUDE.md → seção "Workers → Sheets Worker".

**Service account setup operacional:** Para cada planilha de campanha, o email do service account precisa ter permissão **Editor**. Isto NÃO é responsabilidade do código — fica documentado no painel `/settings` (Story 4.2) e no README.

**Idempotência:** Decisão aberta. Opções:
- A: Coluna F (`order_id`) como chave — antes de append, fazer search por order_id, skip se já existe. Custa 1 read API call por job.
- B: Não preocupar com idempotência (Kiwify retry interno é raro).

Recomendação: **B** no MVP, adicionar A se virar problema real em produção.

**Quota Google:** 60 req/min/user, 300 req/min/projeto. Worker com concurrency=5 e ~1 job/2s fica confortável.
