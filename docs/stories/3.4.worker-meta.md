## Story

**As a** gestor da campanha,
**I want** cada evento configurado disparar um template HSM no WhatsApp do contato,
**so that** o aluno receba comunicação imediata via canal de maior abertura (94%+).

## Acceptance Criteria

1. `src/workers/meta.worker.ts` cria BullMQ Worker consumindo `queue:meta`.
2. Envia template HSM via `POST https://graph.facebook.com/v20.0/{META_PHONE_NUMBER_ID}/messages` (versão configurável).
3. Auth: header `Authorization: Bearer {META_TOKEN}`.
4. Body conforme spec da Meta Cloud API:
   ```json
   {
     "messaging_product": "whatsapp",
     "to": "5541999999999",
     "type": "template",
     "template": { "name": "<template_name>", "language": { "code": "pt_BR" } }
   }
   ```
5. Template = `campaign.meta_templates[event_id]`. Se ausente → skip (não é erro, só não envia).
6. Telefone normalizado para formato E.164 sem `+` (`5541999999999`), reusando helper da Story 3.2.
7. Retry policy padrão (4 tentativas, backoff exponencial).
8. Erros classificados:
   - 401/403 token inválido → fail-fast pra DLQ
   - 429 rate limit → retry com Retry-After respeitado
   - 100 (template pausado/rejeitado pela Meta) → fail-fast (não retry, log claro)
   - 5xx → retry
9. Logs estruturados com correlation_id e `message_id` da Meta na resposta de sucesso.
10. **Suporte a placeholders no template:** Se o template tem parameters, expandir com dados do contato (nome, primeiro_nome, valor_da_compra, etc.). MVP: apenas `{{1}}` = primeiro nome do Customer.
11. Testes cobrem: send OK, 429 com Retry-After, 100 template pausado fail-fast, sem template configurado → skip silencioso, com placeholder.

## Dev Notes

**Referência:** CLAUDE.md → seção "Workers → Meta Worker".

**Rate limit Meta:** 60 msg/min/phone_number padrão (tier mais baixo). O painel exibe tier atual em Settings. Worker deve ter concurrency limitada a ~5 com BullMQ rate limiter.

**Template names:** São registrados no Business Manager da Meta com aprovação manual. O painel apenas referencia pelo nome — não cria templates pela API.

**Idempotência:** Meta não permite enviar 2 templates idênticos pro mesmo número em <24h (rejeita com code 131047 — "Re-engagement"). Comportamento esperado.

**Versão da API:** Usar `v20.0` no MVP, configurável por env (`META_API_VERSION` default `v20.0`).
