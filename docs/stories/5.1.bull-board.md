## Story

**As a** Lucas (operador),
**I want** Bull Board montado em `/queue` no painel, autenticado,
**so that** eu inspecione jobs em tempo real (active/waiting/delayed/completed/failed) sem precisar logar no Redis.

## Acceptance Criteria

1. Bull Board (`@bull-board/api` + `@bull-board/fastify`) servido em `/queue`.
2. As 4 filas (`sheets`, `chatwoot`, `mautic`, `meta`) registradas no Bull Board.
3. Protegido pela mesma Basic Auth do painel — não acessível anonimamente.
4. Visualmente coerente: idealmente um wrapper que mantém sidebar do painel (Bull Board iframe ou rota lateral). MVP aceita: Bull Board com layout default acessado direto em `/queue` (sem sidebar) — usuário usa botão "voltar" do browser.
5. Reprocessar / Retry / Promote / Delete de jobs funcional via Bull Board.
6. Documentação curta em README sobre como usar (link pra docs do Bull Board).

**Referência:** CLAUDE.md → seção "Painel UI — Rotas" linha `/queue`.

**Visual no prototype:** `src/ui/queue.jsx` mostra um Bull Board customizado dentro do tema do painel. Para MVP, **não** replicar essa UI custom — usar Bull Board nativo. A Story 5.2 (Logs/DLQ) cobre a UI custom para o caso específico de reprocessamento da DLQ que precisa ser mais amigável.

**Não duplicar funcionalidade:** Bull Board já tem listagem e retry. A 5.2 foca em:
- DLQ com cards mais ricos (mostra error_msg, contact, allows bulk retry com spread)
- Eventos não-mapeados (não é função do Bull Board)
