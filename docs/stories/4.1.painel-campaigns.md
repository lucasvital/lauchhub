## Story

**As a** Lucas (admin do LaunchHub),
**I want** um painel produtivo (não-prototype) que liste, crie, edite e pause campanhas com fidelidade visual 100% ao design dropped em `src/ui/`,
**so that** eu gerencie campanhas em produção sem editar SQL na mão.

## Acceptance Criteria

1. Stack: Vite + React 18 + TypeScript + Tailwind (com tokens do design system de CLAUDE.md mapeados pro `tailwind.config.ts`).
2. Estrutura em `src/painel/` (frontend produtivo) — `src/ui/` permanece como referência visual mas **não é mais servido**.
3. Rotas implementadas via react-router:
   - `/` → Dashboard com métricas reais (do backend, não mock)
   - `/campaigns` → Lista
   - `/campaigns/new` → Modal de criação
   - `/campaigns/:id` → Detalhe com matrix events × workers
4. **Fidelidade visual:** Pixel-perfect contra `src/ui/LaunchHub.html` em ambos os temas (dark default + light toggle persistido).
5. Componentes core extraídos como reutilizáveis: `<Card>`, `<Badge>`, `<Callout>`, `<Modal>`, `<Toast>` (`<ToastProvider>`), `<WorkerChip>`, `<Sparkline>`, `<BarChart>`, `<WebhookUrl>`, `<StatusBadge>`, `<Sidebar>`.
6. Endpoints backend novos necessários:
   - `POST   /api/login` (valida `ADMIN_USER`/`ADMIN_PASSWORD`, retorna cookie HttpOnly OU JWT)
   - `POST   /api/logout`
   - `GET    /api/me` (verifica sessão)
   - `GET    /api/campaigns` (list, com filtros `q`, `active`)
   - `POST   /api/campaigns`
   - `GET    /api/campaigns/:id`
   - `PATCH  /api/campaigns/:id` (config, enabled_workers, tags, templates)
   - `POST   /api/campaigns/:id/toggle-active`
   - `GET    /api/dashboard/summary` (24h totals, by worker, recent failures)
   - `GET    /api/dashboard/throughput` (24h hourly bars)
7. Auth: **tela de login no React** → `POST /api/login` → cookie HttpOnly (preferido) ou JWT em memória. Middleware Fastify protege `/api/*` exceto `/api/login` e `/webhook/*`. Frontend redireciona pra `/login` em 401.
8. Form de criação valida:
   - Nome obrigatório
   - Token único, regex `^[a-z0-9-]+$`, autogerado do nome (igual ao prototype)
   - Product ID opcional
9. Webhook URL no detalhe da campanha com botão copiar (toast de confirmação).
10. Matrix events × workers toggla cor por worker (verde Sheets, cyan Chatwoot, roxo Mautic, amber Meta) — exatamente como no prototype.
11. Build produção: `npm run build` gera bundle estático pronto pra deploy no Vercel (`dist/` ou padrão Vite).
12. Bundle inicial < 250KB gzipped.
13. **Frontend hospedado no Vercel** (decisão arquitetural 2026-05-14): repo conecta no Vercel, build automático em cada push. Env vars: `VITE_API_BASE_URL=https://launches.loyoladigital.com` (backend Coolify). Webhook do Kiwify continua direto pro backend — não passa pelo Vercel.
14. **CORS no backend:** Fastify libera origens `http://localhost:5173` (Vite dev), `https://*.vercel.app` (previews), e domínio custom Vercel via env `CORS_ALLOWED_ORIGIN`. `credentials: true` obrigatório (cookies cross-origin).

## Dev Notes

**Referência visual canônica:** `src/ui/LaunchHub.html` + arquivos `.jsx`/`.css` na mesma pasta.

**Mapeamento de tokens design → Tailwind:**
```ts
// tailwind.config.ts (snippet)
colors: {
  bg: '#0a0a0f', surface: '#111118', 'surface-2': '#16161f',
  border: '#1e1e2e', 'border-2': '#2a2a3e', dim: '#1a1a2e',
  text: '#e2e8f0', muted: '#64748b', 'muted-2': '#475569',
  accent: { DEFAULT: '#7C3AED', hover: '#6d28d9' },
  'accent-2': '#06b6d4', 'accent-3': '#10b981', 'accent-4': '#f59e0b', 'accent-5': '#ef4444',
}
```

**Anti-pattern:** NÃO usar bibliotecas de UI prontas (shadcn, MUI, Mantine). O design system é proprietário e definido em CLAUDE.md — qualquer biblioteca vai conflitar com tokens e tipografia. Reescrever os componentes do prototype como código produtivo.

**Estado:** React Query (TanStack Query) para cache + invalidação. Zustand opcional para tema/auth. **Não** usar Redux.

**Decisões de @ux-design-expert (quality gate):**
- Acessibilidade WCAG 2.1 AA (foco visível em todos os inputs/buttons, contraste OK em ambos temas, navegação por teclado completa)
- Responsivo mobile-first (prototype tem breakpoints @ 900px e 600px)
