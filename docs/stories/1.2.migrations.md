## Story

**As a** desenvolvedor do LaunchHub,
**I want** migrations versionadas que criam o schema completo do banco (campaigns, global_config, unmatched_events),
**so that** o gateway e o painel possam ler/escrever campanhas e o sistema persista config global e eventos não-mapeados.

## Acceptance Criteria

1. Migration tool escolhida e configurada (`node-pg-migrate` recomendado — alinhar com 1.1).
2. Script `npm run migrate` aplica todas as migrations pendentes; `npm run migrate:down` reverte a última; `npm run migrate:create <name>` gera nova.
3. Migration `001_create_campaigns` cria tabela exatamente conforme CLAUDE.md → seção "Config Store — Schema → campaigns":
   - PK uuid default `gen_random_uuid()`
   - `campaign_token` UNIQUE NOT NULL
   - `product_id` text (nullable)
   - `sheets_id`, `chatwoot_inbox_id`, `chatwoot_tags` jsonb, `mautic_segment_id`, `mautic_tags` jsonb, `meta_templates` jsonb
   - `enabled_workers` jsonb NOT NULL DEFAULT `'{}'` — **(PO validation 2026-05-14, Issue #1):** mapa `event_id → [worker_id...]` configurável por campanha; o gateway (Story 2.1) consulta este campo para decidir quais filas enfileirar
   - `active` boolean default true
   - `created_at`, `updated_at` timestamptz default now()
   - Trigger atualizando `updated_at` em UPDATE
4. Migration `002_create_global_config` cria tabela `global_config(key text PK, value text)`.
5. Migration `003_create_unmatched_events` cria tabela exatamente conforme CLAUDE.md (`id uuid PK`, `payload jsonb`, `token text`, `created_at timestamptz default now()`).
6. Index em `campaigns(campaign_token)` (implícito pela UNIQUE) e index em `unmatched_events(created_at DESC)` para queries do painel.
7. Migration `004_seed_global_config` (opcional) insere chaves esperadas pelo painel com valores vazios: `chatwoot_url`, `mautic_url`, etc.
8. Helpers em `src/db/campaigns.ts`: `findByToken(token)`, `findById(id)`, `list({ active?, query? })`, `create(data)`, `update(id, patch)`, `setActive(id, bool)`, `setEnabledWorkers(id, eventId, workers[])`.
9. Helper em `src/db/unmatched.ts`: `save({ token, payload })`, `list({ limit, offset })`, `delete(id)`.
10. Testes (vitest) cobrem helpers com DB de teste (testcontainers ou DB local descartável).

## Dev Notes

**Schema canônico:** CLAUDE.md → seção "Config Store — Schema". Copiar 1:1.

**Campos jsonb importantes:**
- `chatwoot_tags`: estrutura `{ "<event_id>": ["tag1","tag2"] }`. Mock no prototype `src/ui/data.js` mostra exemplos reais.
- `mautic_tags`: mesma estrutura.
- `meta_templates`: `{ "<event_id>": "template_name" }`.
- `enabled_workers`: `{ "<event_id>": ["sheets","chatwoot",...] }`. **Decisão PO 2026-05-14:** confirmado como campo da tabela. Default empty `{}` — campanhas recém-criadas começam sem workers até serem configuradas no painel. Estrutura exata replicada de `src/ui/data.js`.

**RLS:** Não necessário — banco interno, sem multi-tenant. Auth é via basic auth no painel.

**Greenfield:** Nenhuma migration prévia para reusar. Criar tudo do zero.
